# Internal design substrate

These documents are the project's working design substrate — the architectural
postures, the ADR lifecycle, the development-loop conventions, and the deep
research maps the design draws on. They are kept here, separate from the
product docs, because they describe *how the project thinks about itself* rather
than *how to use antigen*.

If you want to use antigen, start at [`../index.md`](../index.md). If you want
to understand or contribute to its design, these are the references.

## Contents

- [`postures.md`](postures.md) — the seven architectural postures threaded
  through the ADRs.
- [`process.md`](process.md) — the formal ADR lifecycle: proposing, reviewing,
  ratifying, and amending architectural decisions.
- [`dev-loop-ruleset.md`](dev-loop-ruleset.md) — the day-to-day development-loop
  conventions the project builds under.
- [`cross-domain-architectural-map.md`](cross-domain-architectural-map.md) — the
  convergence of 16+ academic fields on the same architectural class antigen
  instantiates.
- [`immune-system-primitive-map.md`](immune-system-primitive-map.md) — the
  comprehensive biology primitive catalog the macro families are recognized
  against.
- [`contact-graph-and-recognition-tiers.md`](contact-graph-and-recognition-tiers.md)
  — the 3-tier × 7-mode recognition framework.

The ratified decisions themselves live in the public
[`../decisions.md`](../decisions.md); this directory holds the surrounding
design rationale and research.
